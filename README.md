# Web Email Client

Веб-клиент для работы с электронной почтой по протоколам IMAP и POP3. Приложение позволяет подключаться к вашему почтовому ящику, просматривать список писем, читать их содержимое и удалять сообщения.

## Используемые технологии
- **Frontend:** React, Tailwind CSS, Lucide React, Vite
- **Backend:** Node.js, Express
- **Работа с почтой:** `imapflow` (для IMAP), `node-pop3` (для POP3), `mailparser` (для парсинга писем)

## Системные требования
Для запуска проекта на вашем локальном компьютере необходимо установить:
- [Node.js](https://nodejs.org/) (рекомендуется версия 18.x или выше)
- npm (обычно устанавливается вместе с Node.js)

## Установка и запуск

1. **Скачайте исходный код проекта** в удобную для вас директорию.

2. **Установите зависимости:**
   Откройте терминал в папке с проектом и выполните команду:
   ```bash
   npm install
   ```

3. **Запуск в режиме разработчика:**
   Для запуска frontend и backend в режиме разработки выполните:
   ```bash
   npm run dev
   ```
   Приложение будет доступно в браузере по адресу: [http://localhost:3000](http://localhost:3000)

4. **Сборка и запуск для продакшена:**
   Если вы хотите собрать оптимизированную версию приложения и запустить её:
   ```bash
   npm run build
   npm start
   ```

## Примечание по авторизации (Пароли приложений)
Для подключения к большинству современных почтовых сервисов (Yandex, Gmail, Mail.ru и др.) использование обычного пароля от аккаунта чаще всего запрещено из соображений безопасности. 
Вам необходимо сгенерировать **Пароль приложения (App Password)** в настройках безопасности вашего почтового провайдера и использовать его для входа в приложение.

## Возможные проблемы и их решение

### Ошибка: `"tsx" не является внутренней или внешней командой`
Эта ошибка означает, что необходимые для разработки пакеты не были установлены. 
1. Убедитесь, что вы выполнили команду `npm install` находясь в корневой папке проекта.
2. Если `npm install` выполнен, но ошибка осталась (иногда бывает на Windows), попробуйте один из следующих вариантов:
   - Запустите команду через npx: `npx tsx server.ts`
   - Или установите пакет tsx глобально: `npm install -g tsx`
